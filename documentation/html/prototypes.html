target_completeness(targlist)[source]¶

Generates completeness values for target stars

Input: targlist - TargetList module object containing target list star attributes. targlist also contains the following available for use: targlist.opt - optical system object targlist.pop - planet population object targlist.rules - rules object targlist.zodi - zodiacal light object targlist.comp - completeness object

Solar system ephemerides class

Observatory.cent(currenttime)[source]¶

Takes current time (MJD) and returns TDB, the time in Julian centuries since the J2000 epoch needed for the algorithm in Vallado

Input: current time (MJD) Output: time in Julian centuries since the J2000 epoch (TDB)

Observatory.distForces(time, targlist, s_ind, secondDist)[source]¶

Finds disturbance forces on occulter and returns values for lateral and axial forces in N

Inputs: time - timekeeping object, targlist - targetlist object, s_ind - index of target star, secondDist - boolean where True is for second occulter

Outputs: dF_lateral - lateral disturbance force, dF_axial - axial disturbance force (both have units u.N)

Observatory.eccanom(M, e)[source]¶

Finds and returns eccentric anomaly given mean anomaly and eccentricity

Inputs: M - mean anomaly, e - eccentricity Output: E - eccentric anomaly

Observatory.keepout(currenttime, catalog)[source]¶

Finds Boolean values for each target at current mission time and returns True if successful

Inputs: current absolute time (astropy Time object), star catalog information Output: True if successful, False if not Finds: Boolean values for each target where True is observable and False is target unobservable (bright object in the keepout zone)

Observatory.keplerplanet(time, planet)[source]¶

Finds heliocentric equatorial position vector (km) for solar system planets using algorithms 2 and 10 from Vallado.

Input: time - current time (astropy Time object), planet - object containing ephemeride data from the SolarEph class Output: r_planet - heliocentric equatorial position vector (km) for solar system planet

Observatory.mass_dec(dF_lateral, t_int)[source]¶

Returns mass_used and deltaV values needed to decrement spacecraft mass for station-keeping

Inputs: dF_lateral - lateral force on occulter (astropy unit N) t_int - integration time (astropy unit day)

Outputs: mass_used - mass used in station-keeping (kg) deltaV - change in velocity required for station-keeping (km/s)

Observatory.moon_earth(time)[source]¶

Finds geocentric equatorial position vector (km) for Earth’s moon using Algorithm 31 from Vallado.

Input: time - current time (astropy Time object) Output: r_moon - geocentric equatorial position vector (km) for Earth’s moon

Observatory.orbit(time)[source]¶

Finds observatory orbit position vector (km) in heliocentric equatorial frame and returns True if successful

Input: current absolute time (astropy Time object) Output: True if successful, False if not Finds: observatory position vector (numpy array) in self.r_sc

This defines the data type expected, orbits are determined by specific Observatory classes

Observatory.propeph(x, TDB)[source]¶

Propagates ephemeride to current time and returns this value

Input: x - ephemeride list (4 elements), TDB - time in Julian centeries since the J2000 epoch Output: y - ephemeride value at current time

Observatory.rot(th, axis)[source]¶

Finds the rotation matrix of angle th about the axis value

Inputs: th - rotation angle, axis - value denoting axis (1,2,3) Output: rotation matrix defined as numpy array

prop_system(r, v, Mp, Ms, dt)[source]¶

Propagates planets’ state vectors (position and velocity) forward in time using the Kepler state transition matrix

Input: r - numpy array containing planet position vectors relative to host stars (astropy unit attached) v - numpy array containing planet velocity vectors relative to host stars (astropy unit attached) Mp - numpy array containing list of planet masses (astropy unit attached) Ms - numpy array containing mass of target star (astropy unit attached) dt - time increment to propagate the system (astropy unit attached)

Output: rnew - propagated position vectors (astropy unit km) vnew - propagated velocity vectors (astropy unit km/day)
