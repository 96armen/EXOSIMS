EXOSIMS.util.keplerSTM module¶

EXOSIMS.util.keplerSTM.pi = 3.141592653589793¶

Kepler State Transition Matrix

Class container for defining a planetary system (or group of planets in multiple systems) via their gravitational parameters and state vectors. Contains methods for propagating state vectors forward in time via the Kepler state transition matrix.

Constructor take the following arguments: x0 - 6n vector of stacked positions and velocities for n planets mu - n vector of standard gravitational parameters mu = G(m+m_s)

where m is the planet mass, m_s is the star mass and G is the gravitational constant
epsmult - default multiplier on floating point precision, used as
convergence metric. Higher values mean faster convergence, but sacrifice precision.

Step function (updateState) takes the following arguments: dt - time step

All units must be complementary (i.e., if position is AU and velocity is AU/day, mu must be in AU^3/day^2.

Algorithm from Shepperd, 1984, using Goodyear’s universal variables and continued fraction to solve the Kepler equation.
